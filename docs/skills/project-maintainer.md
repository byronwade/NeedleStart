# Skill: Project Maintainer

## Purpose

Use this skill for repository-wide stewardship: phase hygiene, task backlog health, governance, risk tracking, and cross-agent collaboration.

## When To Use

- Updating the project backlog or roadmap.
- Reviewing whether a change creates new maintenance obligations.
- Adding collaboration conventions for human and AI contributors.
- Preparing a branch or PR that mainly improves project operations.

## Required Context

Read before editing:

1. `../../AGENTS.md`
2. `../../README.md`
3. `../roadmap.md`
4. `../task-backlog.md`
5. `../risk-mitigation.md`
6. `../phase-1-build-plan.md` for scaffolding or package-boundary changes.

## Workflow

1. Check for unstated assumptions about implemented behavior.
2. Keep Phase 0 focused on constitution and planning unless the task explicitly starts scaffolding.
3. Update backlog entries when new work is introduced.
4. Document rejected approaches for high-impact decisions.
5. Confirm that AI collaboration guidance is vendor-neutral and does not depend on one tool provider.
6. Verify that repository health changes do not introduce executable claims without files and tests.

## Outputs

- Backlog, roadmap, or governance updates.
- Clear next-step recommendations.
- Risk notes for high-risk areas.
