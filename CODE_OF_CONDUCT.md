# Code of Conduct

NeedleStart is an open source project for people building, studying, documenting, testing, and improving app-graph-native React tooling.

We want this community to be useful, welcoming, direct, and safe for contributors at different experience levels.

## Our Standards

Examples of behavior that contributes to a positive environment:

- Be respectful and constructive.
- Assume good intent, but accept correction when impact differs from intent.
- Give actionable technical feedback.
- Keep discussions focused on the work, evidence, and user impact.
- Welcome newcomers and help them find the right docs, issues, and context.
- Credit ideas, bug reports, examples, and research contributions.
- Document disagreements so future contributors understand tradeoffs.

Examples of unacceptable behavior:

- Harassment, intimidation, insults, or personal attacks.
- Demeaning language about a person or group.
- Unwanted sexual attention or imagery.
- Publishing private information without permission.
- Sustained disruption of issues, pull requests, discussions, or community spaces.
- Bad-faith participation, spam, or attempts to derail project governance.

## Scope

This Code of Conduct applies in all project spaces, including:

- GitHub issues, pull requests, discussions, and reviews.
- Documentation and examples.
- Community chat or forum spaces once they exist.
- Public events, talks, demos, and benchmark discussions connected to NeedleStart.

It also applies when someone represents the project publicly.

## Enforcement

Maintainers may take action when behavior violates this Code of Conduct.

Possible actions include:

- Clarifying expectations.
- Editing or removing comments.
- Closing or locking issues or discussions.
- Temporarily limiting participation.
- Banning contributors from project spaces.

Enforcement should be proportionate, documented when appropriate, and focused on keeping the project safe and productive.

## Reporting

Report Code of Conduct concerns privately to the maintainer.

Include:

- What happened.
- Where it happened.
- Who was involved.
- Links or screenshots if available.
- Whether there is immediate safety risk.

Maintainers should handle reports respectfully and avoid unnecessary disclosure.

## Contributor Expectations

NeedleStart values direct technical disagreement. Disagree with ideas, architecture, benchmarks, or tradeoffs. Do not attack people.

Open source works best when sharp thinking and humane collaboration can share the same table.
